# Subscribing to external calendars

The opposite of [Feeds](./feeds): **Subscriptions** let you pull in a
read-only external calendar — someone else's public `.ics` feed, like a
public holiday calendar — and have it stay automatically in sync.

From **Subscriptions → Add**, either:
- Pick one of the built-in **holiday presets** for a one-click setup, or
- Enter the external calendar's URL yourself, choose a slug and
  optional display name/color, and pick how often to check for updates
  (from hourly up to every 90 days).

The subscriptions list shows each one's source, color, and last-sync
status (including the error message if a sync failed). Unsubscribing
removes the local calendar and stops future syncs. You can also reach
this from the calendar page's **Add calendar → Subscribe** menu.
