# Meeting scheduling & invite responses

When you add attendees to an event (see [Calendars](./calendars)):

- **Attendees who are also users on this server** get access to the
  event automatically (they'll see it show up once shared) — respond
  to it using your calendar app's normal accept/decline controls, the
  same way you'd RSVP to any invite.
- **Attendees outside this server** receive a regular **email
  invitation** they can accept or decline from their own email/calendar
  app, exactly like an Outlook or Google Calendar invite. Their reply
  comes back automatically and updates the event's attendee status —
  you don't need to do anything to receive it.

If you cancel an event, attendees are notified the same way (in-app
for local users, email for everyone else).
