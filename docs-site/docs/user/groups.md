# Groups

Groups let you share with a whole team at once instead of granting
access to each person individually. From **Groups**, create a group
with a name, add members, and then grant that group access to a
calendar or address book (see [Sharing](./sharing)) exactly like you
would an individual user — every member inherits the access. If your
organization uses single sign-on, an admin can configure a group to
auto-sync its membership from your identity provider's group claims;
members added this way are marked as auto-assigned.
